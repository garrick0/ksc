# Combining Kinds

KindScript kinds can enforce **multiple properties at once**. A single kind can combine `immutable`, `noConsole`, `noMutation`, and more — catching several categories of issues simultaneously.

## The problem

Open `src/service.ts`. This module defines utility functions annotated with `StrictValue` — a kind that requires immutable bindings, no console usage, and no mutation.

The code has three different kinds of violations:
1. A `let` binding (should be `const`)
2. A `console.log` call
3. Assignment operators building an object imperatively

## Detect the issues

Run the check:

```
npm run check
```

You should see violations from **three different properties** — `immutable`, `noConsole`, and `noMutation` — all enforced by the same `StrictValue` kind.

## Understand the rule

In `src/kinds.ts`:

```ts
type StrictValue = Kind<{ immutable: true; noConsole: true; noMutation: true }>;
```

Each property in the `PropertySet` is checked independently. A single kind can combine any subset of the 8 available properties. When a value is annotated with `StrictValue`, KindScript checks all three constraints on every node in the function body.

## Fix it

**`buildEndpoint`** — change `let` to `const` and remove the `console.log`:
```ts
export const buildEndpoint: StrictValue & ((base: string, path: string) => string) = (base, path) => {
  const url = `${base}/${path}`;
  return url;
};
```

**`buildHeaders`** — use an object literal instead of imperative assignment:
```ts
export const buildHeaders: StrictValue & ((token: string) => Record<string, string>) = (token) => {
  return {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json',
  };
};
```

Run `npm run check` to confirm **0 violations**.

:::tip
Combining properties lets you define domain-specific quality levels. A `StrictValue` in a config module has different constraints than a `Pure` function in a math library — but both are enforced the same way.
:::

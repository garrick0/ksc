# Immutable Bindings

Like ESLint's `prefer-const` rule, KindScript can enforce that code inside annotated functions uses `const` instead of `let` or `var`.

## The problem

Open `src/config.ts`. These string-building functions are annotated with `Immutable`, but they use `let` for local variables that are never reassigned — they should all be `const`.

## Detect the issues

Run the check:

```
npm run check
```

You should see **6 violations** — each `let` binding inside the annotated functions is flagged with a message like `'let' binding violates Immutable (immutable)`.

## Understand the rule

In `src/kinds.ts`, `Immutable` is defined as:

```ts
type Immutable = Kind<{ immutable: true }>;
```

The `immutable` property checks `VariableDeclarationList` nodes inside the annotated function body. If a binding uses `let` or `var` instead of `const`, it's a violation. This catches unnecessary mutability — if a variable is never reassigned, it should be `const`.

## Fix it

Change all `let` declarations inside the three functions to `const`:

```ts
export const buildGreeting: Immutable & ((name: string, title: string) => string) = (name, title) => {
  const greeting = `Hello, ${title} ${name}`;
  const suffix = '!';
  return greeting + suffix;
};
```

Do the same for `formatPrice` and `buildList`. Run `npm run check` to confirm **0 violations**.

:::tip
`immutable` enforces `const` bindings inside annotated functions. If a variable is never reassigned, there's no reason to use `let`. This makes code intent clearer and prevents accidental mutations.
:::

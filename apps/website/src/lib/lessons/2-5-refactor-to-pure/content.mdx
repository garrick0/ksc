# Refactor to Pure

This lesson combines everything from the previous tutorials. You'll fix a module with multiple violation types — imports, console usage, and mutation — to make its functions truly pure.

## The problem

Open `src/analytics.ts`. These computation functions are annotated with `Pure`, a kind that enforces four properties at once: `noImports`, `noConsole`, `noMutation`, and `immutable`.

The code has several issues:
- `computeAverage` uses `+=` to accumulate (mutation)
- `computeTotal` uses `+=` and `console.log` (mutation + console)
- `formatSummary` calls an imported function (violates noImports)

## Detect the issues

Run the check:

```
npm run check
```

You should see violations from multiple properties. Notice how `Pure` catches issues that individual properties would catch separately.

## Understand the kind

In `src/kinds.ts`:

```ts
type Pure = Kind<{ noImports: true; noConsole: true; noMutation: true; immutable: true }>;
```

A `Pure` function cannot:
- Use any imported bindings (`noImports`)
- Access `console.*` (`noConsole`)
- Use assignment operators or `++`/`--` (`noMutation`)
- Use `let` or `var` (`immutable`)

This is a strict constraint — the function can only use its parameters, local `const` bindings, and built-in methods.

## Fix it

**`computeAverage`** — replace the `+=` accumulator with `reduce`:
```ts
export const computeAverage: Pure & ((values: number[]) => number) = (values) => {
  return values.length > 0
    ? values.reduce((acc, v) => acc + v, 0) / values.length
    : 0;
};
```

**`computeTotal`** — replace `+=` with `reduce` and remove `console.log`:
```ts
export const computeTotal: Pure & ((items: { price: number; qty: number }[]) => number) = (items) => {
  return items.reduce((acc, item) => acc + item.price * item.qty, 0);
};
```

**`formatSummary`** — inline the logic instead of importing `formatEvent`:
```ts
export const formatSummary: Pure & ((name: string, value: number) => string) = (name, value) => {
  return `${name}: ${value.toFixed(2)}`;
};
```

Run `npm run check` to confirm **0 violations**.

:::tip
Pure functions are the gold standard for testable, predictable code. They depend only on their inputs and produce no side effects. KindScript's `Pure` kind makes this contract explicit and enforceable.
:::

4. Key Internal Data Structures

// The internal Type object (NOT the AST node)
interface TypeParameter extends Type {
  kind: TypeFlags.TypeParameter;
  constraint?: Type;          // resolved constraint (not AST node)
  default?: Type;             // resolved default type
  target?: TypeParameter;     // for instantiated type params
  mapper?: TypeMapper;        // for instantiated type params
  isThisType?: boolean;       // is this the implicit 'this' param
  symbol: Symbol;             // link back to declaration
}

Critical distinction: the AST node TypeParameterDeclaration is syntax (text position, constraint as TypeNode). The Type object TypeParameter is semantic (resolved constraint as a Type, used during checking). The checker creates one from the other during type resolution.

// A TypeMapper is a function, not a data structure
type TypeMapper = (t: TypeParameter) => Type | undefined;

// Simple mapper: replace T with string
const simpleMapper = (t) =>
  t === T ? stringType : undefined;

// Composed mapper: nested generics
const composedMapper = (t) =>
  outerMapper(t) ?? innerMapper(t);

// Array mapper: replace [T, U] with [string, number]
const arrayMapper = (t) => {
  const idx = typeParams.indexOf(t);
  return idx >= 0 ? typeArgs[idx] : undefined;
};

Mappers are functions because they need to compose efficiently. When you have Outer<Inner<T>>, the checker composes the outer and inner mappers into a single function rather than building an intermediate data structure.

// Created when calling a generic function without explicit types
interface InferenceContext {
  signature: Signature;
  typeParameters: TypeParameter[];
  inferences: InferenceInfo[];     // one per type param
  flags: InferenceFlags;
}

interface InferenceInfo {
  typeParameter: TypeParameter;
  candidates: Type[];              // from covariant sites
  contraCandidates: Type[];        // from contravariant sites
  inferredType?: Type;             // final result
  priority: InferencePriority;
  isFixed: boolean;
}

Inference is two-pass: first pass processes all arguments except callbacks (to establish context), second pass processes callbacks with the partial context. This is why arr.map(x => x.name) works — the first pass infers the array element type, then the second pass uses it to type the callback parameter.

// Instantiation cache prevents redundant work
// Key: (genericType, typeArguments tuple)
// Value: instantiated type

Map<string, Type> instantiationCache;
// "Array|string" → Array<string> (already computed)
// "Map|string|number" → Map<string, number>

// Safety limits:
// instantiationCount: number (max ~5,000,000)
// instantiationDepth: number (max ~100)

// When limits are hit:
// "Type instantiation is excessively deep
//  and possibly infinite. ts(2589)"

Without caching, DeepPartial<BigType> would re-instantiate every nested property type. With caching, each unique (Type, Mapper) pair is computed once. The 5M limit prevents pathological cases like deeply recursive conditional types from crashing the compiler.

// A GenericType is the "template" — uninstantiated
interface GenericType extends ObjectType {
  typeParameters: TypeParameter[];  // [T] or [T, U]
  instantiations: Map;              // cached instances
}

// Array is a GenericType with typeParameters: [T]
// Array<string> is an instantiation of Array with T=string

// When you write Array<string>:
//   1. Look up GenericType for Array
//   2. Check instantiations cache for T=string
//   3. If not cached: instantiate, cache, return
//   4. If cached: return immediately

The GenericType holds the template. Each instantiation is a separate Type object that references its GenericType parent. This allows structural comparison: two Array<string> from different files resolve to the same instantiated type.

5. Advanced: Conditional Type Distribution

How distribution works internally

type ToArray<T> = T extends any ? T[] : never;

// ToArray<string | number>
// Step 1: Is checkType a naked type parameter? YES (T)
// Step 2: Is the argument a union? YES (string | number)
// Step 3: DISTRIBUTE — apply to each member:
//   ToArray<string> = string extends any ? string[] : never = string[]
//   ToArray<number> = number extends any ? number[] : never = number[]
// Step 4: Union the results:
//   string[] | number[]

// Compare WITHOUT distribution:
type ToArray2<T> = [T] extends [any] ? T[] : never;
// ToArray2<string | number>
// checkType is [T] (NOT a naked parameter) → no distribution
// [string | number] extends [any] ? (string | number)[] : never
// = (string | number)[]   ← single array type, not union!

The checker's getConditionalType() function checks: is the checkType a type parameter that maps to a union? If so, it iterates over each union member, evaluates the conditional for each, and unions the results. This is why Exclude<T, U> = T extends U ? never : T works — it filters unions member by member.

6. Advanced: Variance Checking

// TypeScript 4.7+ explicit variance annotations:
interface Producer<out T> {    // T appears only in output positions
  get(): T;
}

interface Consumer<in T> {     // T appears only in input positions
  accept(value: T): void;
}

interface Invariant<in out T> { // T appears in both positions
  transform(value: T): T;
}

// Without annotations, the checker INFERS variance by scanning
// where T appears in the type's structure:
//   Return types → covariant (out)
//   Parameter types → contravariant (in)
//   Both → invariant (in out)
//   Neither → independent (bivariant in practice)

Why variance matters for performance: When checking Producer<Cat> assignable to Producer<Animal>, the checker can skip the full structural comparison and just check Cat extends Animal. Without variance annotations, it must structurally compare every member.

How Effect-TS uses this: Effect<out A, out E, out R> declares all three parameters covariant. This tells the checker: if Effect<Cat, NotFound, Db> is needed, Effect<Animal, NotFound | Timeout, Db | Logger> works because each parameter can be widened independently.

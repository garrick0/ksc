Underserved Markets

Sectors where formal verification would deliver significant value but adoption is minimal or nonexistent. These represent the greenfield opportunity — markets where an accessible, cost-effective tool could create entirely new demand.

The Core Gap: Regulatory Intent vs Tooling Reality

Pattern across all underserved markets: Australian regulations (SOCI Act, APRA CPS 234, ASD ISM, AESCSF) increasingly mandate "software assurance" and "secure development" — but none explicitly require static analysis or formal verification. They specify the outcome (secure software) without specifying the method. This creates a gap: organizations know they need better software assurance but lack tooling mandates to justify procurement. A vendor that can map tool capabilities to regulatory outcomes has a significant first-mover advantage.

The Fortescue incident (2019): An autonomous truck-to-truck collision at Christmas Creek mine was caused by a Wi-Fi communications software error. These are 240-tonne vehicles operating 24/7 in remote areas. The consequences of a software bug in this domain are catastrophic — and the incident demonstrates that testing alone is insufficient. What Miners Would Care About

Safety

Prove absence of runtime errors in autonomous vehicle control software. Verify communication protocol correctness. Prevent another Fortescue-type incident. Each incident risks multi-million dollar damage + reputational harm + regulatory scrutiny.

Uptime / Productivity

Every autonomous truck that stops = lost production. Proving software correctness reduces unplanned stops from software faults. At $100K+/hour for a haul truck fleet, even small improvements in uptime are worth millions annually.

Regulatory De-risking

ISO 17757 compliance. Safety case evidence. Demonstrating "state of the art" verification to regulators. Proactive verification is cheaper than reactive incident response + investigation + remediation.

Pitch to miners: "You're operating 240-tonne vehicles autonomously at scale, supervised from 1,200 km away. Your current verification is testing-based. We can mathematically prove your control software is free of the class of bugs that caused the 2019 Fortescue collision — communication errors, race conditions, overflow bugs. The cost of our tool is less than one hour of fleet downtime."

Regulated Industries — Telco, Banking, Energy

Post-breach Australia has created a new regulatory reality. The Optus breach, Medibank hack, and SOCI Act expansion mean software quality is now a board-level concern across multiple sectors.

▶ Telecommunications — Post-Breach Opportunity Urgent
The Optus breach (2022): An access control coding error in an API exposed 10 million customers' personal data. This was not a sophisticated attack — it was a simple software bug. Static analysis or formal verification of the API code would have caught it. Remediation costs are in the hundreds of millions. The coding error was so basic that it would have been flagged by even mid-range static analysis tools.
Optus 2024 — Second Breach
Software flaw in identity verification provider enabled SIM-swap fraud. Again, a software bug — not a sophisticated attack.
